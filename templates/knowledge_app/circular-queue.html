🔄 循环队列 (Circular Queue)

解决假溢出问题的智能队列，充分利用数组空间

📋

知识点概述

核心概念：循环队列是顺序队列的改进版本，通过将数组首尾相连形成环形结构，使用取模运算实现指针的循环移动，彻底解决了"假溢出"问题。

🔄 循环特性

指针到达数组末尾后自动回到开头

📐 取模运算

index = (index + 1) % capacity

❌ 假溢出解决

充分利用所有可用空间

🎯 满空判断

预留一个空位区分满和空状态

🧮 核心公式

入队：rear = (rear + 1) % maxSize
出队：front = (front + 1) % maxSize
队满：(rear + 1) % maxSize == front
队空：front == rear

高级前置知识：顺序队列、取模运算、数组操作

🔍

概念详解

🎠 生活类比：旋转木马

想象一个旋转木马游乐设施：

  • 木马座位 = 数组元素（固定数量的位置）
  • 上马点 = rear指针（新游客上马的位置）
  • 下马点 = front指针（游客结束游戏下马的位置）
  • 旋转 = 循环特性（到最后一个位置后回到第一个）
  • 预留位置 = 空余座位（区分满载和空转状态）

💡 关键：木马永远在转，没有"死角"或浪费的位置！

⚡ 问题与解决方案

❌ 顺序队列的假溢出

  • 数组前面有空位
  • rear指针已到末尾
  • 无法继续入队
  • 空间利用率低

[_, _, 3, 4, 5] ← rear已满？

✅ 循环队列的解决

  • 指针循环移动
  • 充分利用空间
  • 真正的满才停止
  • 空间利用率高

[6, 7, 3, 4, 5] ← 循环利用

🔑 关键理解点：
  • 为什么预留一位？ 如果不预留，当front==rear时无法区分是空还是满
  • 实际容量 = 数组大小 - 1（牺牲一个位置换取清晰的状态判断）
  • 取模运算 是实现循环的核心，确保指针在有效范围内循环
  • 状态判断 比顺序队列复杂，需要理解循环逻辑

#include <iostream>
using namespace std;

class CircularQueue {
private:
    int* arr;
    int front, rear;
    int maxSize;      // 数组实际大小
    int capacity;     // 可用容量 = maxSize - 1

public:
    CircularQueue(int size) {
        maxSize = size + 1;  // 多分配一位用于区分满空
        capacity = size;
        arr = new int[maxSize];
        front = rear = 0;
    }

    ~CircularQueue() {
        delete[] arr;
    }

    // 入队操作
    bool enqueue(int data) {
        if (isFull()) {
            cout << "队列已满!" << endl;
            return false;
        }

        arr[rear] = data;
        rear = (rear + 1) % maxSize;  // 循环移动
        cout << "入队成功: " << data << endl;
        return true;
    }

    // 出队操作
    int dequeue() {
        if (isEmpty()) {
            cout << "队列为空!" << endl;
            return -1;
        }

        int data = arr[front];
        front = (front + 1) % maxSize;  // 循环移动
        cout << "出队成功: " << data << endl;
        return data;
    }

    // 查看队头元素
    int peek() {
        if (isEmpty()) {
            cout << "队列为空!" << endl;
            return -1;
        }
        return arr[front];
    }

    // 判断队列是否为空
    bool isEmpty() {
        return front == rear;
    }

    // 判断队列是否已满
    bool isFull() {
        return (rear + 1) % maxSize == front;
    }

    // 获取队列大小
    int size() {
        return (rear - front + maxSize) % maxSize;
    }

    // 显示队列状态
    void display() {
        if (isEmpty()) {
            cout << "队列为空" << endl;
            return;
        }

        cout << "队列内容: ";
        int i = front;
        while (i != rear) {
            cout << arr[i] << " ";
            i = (i + 1) % maxSize;
        }
        cout << endl;
        cout << "front=" << front << ", rear=" << rear << endl;
    }

    // 获取实际容量
    int getCapacity() {
        return capacity;
    }
};
public class CircularQueue {
    private int[] arr;
    private int front, rear;
    private int maxSize;      // 数组实际大小
    private int capacity;     // 可用容量

    public CircularQueue(int size) {
        this.maxSize = size + 1;  // 多分配一位用于区分满空
        this.capacity = size;
        this.arr = new int[maxSize];
        this.front = this.rear = 0;
    }

    // 入队操作
    public boolean enqueue(int data) {
        if (isFull()) {
            System.out.println("队列已满!");
            return false;
        }

        arr[rear] = data;
        rear = (rear + 1) % maxSize;  // 循环移动
        System.out.println("入队成功: " + data);
        return true;
    }

    // 出队操作
    public int dequeue() {
        if (isEmpty()) {
            System.out.println("队列为空!");
            return -1;
        }

        int data = arr[front];
        front = (front + 1) % maxSize;  // 循环移动
        System.out.println("出队成功: " + data);
        return data;
    }

    // 查看队头元素
    public int peek() {
        if (isEmpty()) {
            System.out.println("队列为空!");
            return -1;
        }
        return arr[front];
    }

    // 判断队列是否为空
    public boolean isEmpty() {
        return front == rear;
    }

    // 判断队列是否已满
    public boolean isFull() {
        return (rear + 1) % maxSize == front;
    }

    // 获取队列大小
    public int size() {
        return (rear - front + maxSize) % maxSize;
    }

    // 显示队列状态
    public void display() {
        if (isEmpty()) {
            System.out.println("队列为空");
            return;
        }

        System.out.print("队列内容: ");
        int i = front;
        while (i != rear) {
            System.out.print(arr[i] + " ");
            i = (i + 1) % maxSize;
        }
        System.out.println();
        System.out.println("front=" + front + ", rear=" + rear);
    }

    // 获取实际容量
    public int getCapacity() {
        return capacity;
    }
}
class CircularQueue:
    """循环队列类"""

    def __init__(self, size):
        self.max_size = size + 1  # 多分配一位用于区分满空
        self.capacity = size      # 实际可用容量
        self.arr = [None] * self.max_size
        self.front = 0
        self.rear = 0

    def enqueue(self, data):
        """入队操作"""
        if self.is_full():
            print("队列已满!")
            return False

        self.arr[self.rear] = data
        self.rear = (self.rear + 1) % self.max_size  # 循环移动
        print(f"入队成功: {data}")
        return True

    def dequeue(self):
        """出队操作"""
        if self.is_empty():
            print("队列为空!")
            return None

        data = self.arr[self.front]
        self.front = (self.front + 1) % self.max_size  # 循环移动
        print(f"出队成功: {data}")
        return data

    def peek(self):
        """查看队头元素"""
        if self.is_empty():
            print("队列为空!")
            return None
        return self.arr[self.front]

    def is_empty(self):
        """判断队列是否为空"""
        return self.front == self.rear

    def is_full(self):
        """判断队列是否已满"""
        return (self.rear + 1) % self.max_size == self.front

    def size(self):
        """获取队列大小"""
        return (self.rear - self.front + self.max_size) % self.max_size

    def display(self):
        """显示队列状态"""
        if self.is_empty():
            print("队列为空")
            return

        result = []
        i = self.front
        while i != self.rear:
            result.append(str(self.arr[i]))
            i = (i + 1) % self.max_size

        print("队列内容:", " ".join(result))
        print(f"front={self.front}, rear={self.rear}")

    def get_capacity(self):
        """获取实际容量"""
        return self.capacity

# 使用示例
if __name__ == "__main__":
    # 创建容量为5的循环队列
    queue = CircularQueue(5)

    # 测试入队
    for i in range(1, 6):
        queue.enqueue(i * 10)

    queue.display()

    # 测试出队
    queue.dequeue()
    queue.dequeue()

    # 测试循环特性
    queue.enqueue(60)
    queue.enqueue(70)

    queue.display()

    print(f"队列大小: {queue.size()}")
    print(f"队头元素: {queue.peek()}")
    print(f"是否已满: {queue.is_full()}")
    print(f"是否为空: {queue.is_empty()}")

🔍 深度案例：生产者-消费者问题

循环队列是解决生产者-消费者问题的经典数据结构：

  • 🏭 生产者：不断向队列添加产品（入队操作）
  • 🛒 消费者：不断从队列取出产品（出队操作）
  • 🔄 循环特性：充分利用有限的缓冲区空间
  • ⚡ 高效处理：O(1)的操作时间，适合高频访问
  • 🎯 状态控制：清晰的满/空状态判断，避免竞态条件

💡 选择指南

🎯 适用场景

  • 缓冲区大小固定且已知
  • 需要高效的空间利用率
  • 频繁的入队出队操作
  • 实时系统对性能要求高

⚠️ 不适用场景

  • 队列大小变化很大且不可预测
  • 需要随机访问队列中的元素
  • 对内存使用极度敏感的场景
  • 需要动态扩容的应用

学习检验

🤔 问题1：为什么循环队列需要预留一个空位？

答案：为了区分队列满和队列空的状态
详细解析：
  • 如果不预留：当数组完全填满时，front == rear
  • 队列为空时：front == rear
  • 无法区分：同样是front == rear，但含义完全不同
  • 预留一位后：满队列时 (rear+1)%maxSize == front，空队列时 front == rear

🤔 问题2：循环队列中如何计算实际元素个数？

答案：size = (rear - front + maxSize) % maxSize
解析：
  • 为什么加maxSize？ 防止rear < front时出现负数
  • 情况1：rear >= front，如front=2, rear=5, size=3
  • 情况2：rear < front（循环了），如front=7, rear=2, size=3
  • 公式验证：(2-7+8)%8=3 ✓

🤔 问题3：给定maxSize=6的循环队列，依次入队1,2,3,4,5，然后出队2次，再入队6,7，此时front和rear的值？

答案：front = 2, rear = 1
步骤分析：
  • 初始：front = 0, rear = 0
  • 入队1,2,3,4,5后：front = 0, rear = 5，数组[1,2,3,4,5,_]
  • 出队2次后：front = 2, rear = 5，数组[_,_,3,4,5,_]
  • 入队6：rear = (5+1)%6 = 0，数组[6,_,3,4,5,_]
  • 入队7：rear = (0+1)%6 = 1，数组[6,7,3,4,5,_]
  • 最终：front = 2, rear = 1

🤔 问题4：循环队列相比顺序队列的主要优势是什么？

主要优势：
  • 解决假溢出：充分利用数组的每个位置
  • 空间效率高：不会因为front指针前移而浪费空间
  • 操作一致：入队出队时间复杂度都是O(1)
  • 适合缓冲：固定大小的缓冲区应用场景理想
代价：实现稍复杂，需要理解取模运算和状态判断

🤔 问题5：什么情况下应该选择循环队列而不是链队列？

选择循环队列的场景：
  • 固定容量：队列大小上限明确且稳定
  • 高频操作：需要极高的入队出队性能
  • 内存连续：缓存友好，提高访问效率
  • 无动态分配：避免频繁的内存分配和释放
  • 嵌入式系统：内存受限，需要精确控制内存使用
选择链队列的场景：队列大小变化大、需要动态扩容
